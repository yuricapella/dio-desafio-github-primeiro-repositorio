existem os 3 conceitos do git:
Unmodified - Modified - Staged

unmodified: é um arquivo que acabou de ser criado, nao foi mexido (logo nao existe de certa forma)
modified: é um arquivo que acabou de ser modificado, houve alguma alteração nele.
Staged: é quando usamos o comando git add * para prepara-lo a um commmit.

git add arquivo (Prepara o arquivo para ser salvo em um commit basicamente (STAGED))
git add * (Prepara todos os arquivos do repositório para serem salvos em um commit (STAGED))
git commit -m "comentario" (commit da um significado a alteração do arquivo, salva ele, mostra quem alterou, oque foi alterado, o quanto foi alterado.)
basicamente, sempre que vc fazer um arquivo (unmodified) e altera-lo (modified), vc usa o GIT ADD para prepara-lo (staged) para um commit.
Após usar o commit, o arquivo volta ao estado (unmodified, que basicamente é como se você acabasse de criar um novo arquivo com aqueles dados salvos) e assim sucessivamente vai usando os comandos, é um ciclo.
